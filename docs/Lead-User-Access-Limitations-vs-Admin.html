Lead User Access Limitations vs Admin

DentraCRM – What access is limited for the Lead User role compared to Admin.

Open this file in Microsoft Word and use File → Save As → Word Document (.docx) if you need a .docx file.

1. Summary

  • Lead User can only see and manage leads assigned to them. Admin can see all leads across all clinics.
  • Lead User cannot access: Analytics, User management, TBD (unassigned) leads queue, Lead assignment/reassignment, Delete lead, Export leads, or Reports exports. These are Admin only.
  • Lead User can add leads, edit assigned leads (including clinic and status), schedule/cancel appointments, and add notes (but cannot create “Admin only” notes).

2. Access Limited to Lead User (Admin-Only Features)

The following are only available to Admin (and Super Admin). Lead Users do not have access.

Area What is restricted (Admin only)
Leads – Visibility Lead User sees only leads assigned to them. Admin sees all leads across all clinics.
Leads – Unassigned (TBD) queue The “Unassigned Leads (TBD)” section on the Leads page is Admin only. Lead User does not see it.
Leads – Assign / Reassign Dedicated “Assign lead to user” / reassign flow is Admin only. Lead User can change clinic for their assigned leads via edit form or clinic dropdown, but cannot assign or reassign the lead to another user.
Leads – Delete Delete lead is Admin only. Lead User cannot delete leads.
Leads – Export Export leads (e.g. CSV/Excel) is Admin only.
Leads – Status history Viewing lead status history (who changed status and when) is Admin only. Lead User cannot see this.
Leads – Assigned user in list “Assigned user” column/details in lead list and APIs are returned only for Admin. Lead User does not receive assigned-user data in list responses.
Leads – Filter by clinic Filtering leads by clinic in the Filters panel is Admin only. Lead User does not see the clinic filter.
Analytics Entire Analytics section (dashboard, funnel, by clinic, by source) is Admin only. Lead User does not see the Analytics menu or pages.
Users Entire Users section (list, create, edit, delete users, invite) is Admin only. Lead User does not see the Users menu or pages.
Reports – Exports Report exports (e.g. DNC/DNR export, clinic export, full export) are Admin only. Lead User can view reports in the app but cannot run export endpoints.
Schedule appointment – clinic picker When scheduling an appointment, choosing “Schedule at clinic” (a different clinic than the lead’s current one) is Admin only. Lead User schedules at the lead’s current clinic only.
Notes – “Admin only” notes Only Admin can create notes marked “Admin only” (hidden from clinic staff). Lead User can add notes but cannot mark them as Admin only.

3. What Lead User Can Do (Shared with Admin)

Lead User has the following access (same or similar to Admin, but only for assigned leads where applicable).

Area Lead User access
Leads – List & detail View leads list and lead detail (only leads assigned to them).
Leads – Add Create new leads (Add Lead). New leads can be auto-assigned to the creating Lead User or first Lead User depending on tenant setup.
Leads – Edit Edit assigned leads: name, phone, email, clinic, status, priority, follow-up, next action, etc. Cannot edit leads not assigned to them.
Leads – Clinic Change clinic for assigned leads (via edit form or clinic dropdown). Cannot assign lead to another user.
Leads – Status & priority Update status and priority for assigned leads (within allowed statuses for Lead User: e.g. Visited, Treatment, Tx Denied, Rescheduled, Lost, DNA, DNR, TWC).
Appointments View and manage appointments for their assigned leads: schedule, reschedule, cancel. Cannot choose “Schedule at clinic” for a different clinic (Admin only).
Notes Add and view notes on assigned leads. Cannot create “Admin only” notes.
Reports – View View Reports pages (DNC/DNR, clinic, full). Cannot run export endpoints.
Patient journey tabs Access Appointments, Visited, Treatment, Tx Denied, Follow-ups, DNR/DNC, Lost (for their assigned leads / relevant data).
Settings Access Settings (all roles).

Note: “Assigned leads” means leads whose Assigned User is the current Lead User. Assignment is set by Admin (or by system when a Lead User creates a lead, depending on tenant configuration).
